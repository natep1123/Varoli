## 1. Purpose

This document formalizes the leading candidate solution for structural reference in Varoli.

Reference is the problem of how a later resolution unit can point back to a specific stable world established earlier in a chain. Without a reference mechanism, every stable world is anonymous: the form `aVa` in a later unit is structurally indistinguishable from a first use of that form. The language needs a way to say "the aVa that was established, not a new one."

The current audit identified this as Barrier 1. The frontier priorities document (now updated) listed it as Priority 4. The proposed solution requires no new primitives, no new classes, and no reinterpretation of existing structural properties. It derives from binary directionality already established in the architecture.

**Authority:** This is a working hypothesis. The core mechanism is architecturally motivated and consistent with all current structural principles. It has not been independently contrast-tested. The document is honest about this status throughout.

---

## 2. The core proposal

When a stable world has been established via a CV arrival (e.g., `va` arriving into `aVa`), the VC departure face of the same attractor pair (`av`) serves as a structural reference to that form in subsequent units.

**CV builds.** A CV arrival form establishes a stable world as its closure target. `va` arriving into `aVa` completes: the living force arrives at the relational domain and closes within the centered relational stable world.

**VC references.** After the stable world exists, the VC departure face of the same attractor pair - the reverse orientation - refers to that established form rather than establishing a new one. `av` in a post-establishment context is not a fresh outward force. It is the relational domain expressing through vitality in reference to the form already established.

The mechanism is structural recognition: the departure face is the non-equivalent complement of the arrival face (Law 1). In a post-establishment context where the stable world is already present, the departure face is the natural structure for referencing that world.

---

## 3. Derivation from binary directionality

Binary directionality (Law 1) states that CV and VC are non-equivalent and encode structurally distinct realities: arrival and departure, inward and outward. This non-equivalence is the structural foundation of reference.

If CV builds a stable world by arrival, and if CV and VC are non-equivalent, then VC in a context where the stable world already exists cannot perform the same building operation. It must do something structurally distinct. The most natural and architecturally grounded reading of VC in a post-establishment context is that it references the form the arrival built - it is the departure face of the same centered structure.

This is not an extra mechanism layered onto the architecture. It is the structural consequence of binary directionality applied to a context where one orientation has already established a stable world. The non-equivalence that made CV and VC structurally distinct for building is the same non-equivalence that makes the VC face structurally recognizable as referring to the CV-built form.

## 5. Three reference depths

The reference mechanism has three depths of specificity, each derivable from the core proposal.

### 5.1 Departure face of the attractor pair

`av` after `aVa` is established = generic structural reference to the relational-vitality form. The reference is anchored to the specific attractor pair. It identifies the form by the same kinetic-substrate relationship that established it, viewed from the departure direction.

### 5.2 Bare attractor center

`V` (bare vitality, where capitalization marks the attractor center) or `oRo` (bare resonant earth) as a compressed reference to the full expanded form. This is consistent with the center-governed interpretation (Law 3): the attractor alone is sufficient to identify the stable form in contexts where the full form is already established.

### 5.3 Qualified departure face

`avz` (departure face with seeking qualification) or `avs` (departure face with directional probe) as a locked or searching reference. The departure face with suffix qualification identifies a specific instance of the form with additional resolution. This depth is provisional and downstream - it requires the qualified departure face to behave consistently with the qualification architecture before it can be confirmed.

## 6. What this does not yet answer

**Disambiguation between multiple instances.** If `aVa` has been established multiple times in a chain, the departure face alone may not distinguish which instance is being referenced. Whether temporal or positional information can resolve this without additional mechanism is not yet determined.

**First-use detection.** The proposal requires that VC be distinguishable as reference rather than as a new outward force. The mechanism for this detection is contextual: the stable world must already be present in the chain for the reference reading to activate. Whether this is sufficient or whether explicit establishment markers are required is an open question.

**Reference to non-canonical stable forms.** The current examples use attractor-pair stable worlds (`aVa`, `arOva`). Whether the departure-face mechanism applies to general-consonant stable worlds (e.g., `aNa` established, `an` referencing) is not tested.

**Scope across longer chains.** Whether a departure face continues to reference a stable world across many intervening units or whether reference scope fades is not yet determined.

---

## 7. Honest status marking

This is a working hypothesis. The core mechanism is the strongest candidate solution for reference in the current architecture. It is architecturally motivated, derives from established structural principles, and requires no new primitives.

It should not be treated as established until:

- contrast-testing confirms that VC departure forms in post-establishment contexts are read as references rather than as independent outward forces
- the disambiguation question is addressed for chains containing multiple instances of the same form
- at least one extended chain is formally constructed using the reference mechanism and confirmed coherent

Until those conditions are met, use this document as the architectural candidate and mark any downstream use of the mechanism as provisional.
