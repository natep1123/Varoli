# 00 — Varoli Ground 4 Grammar Architecture Overview

*Last Updated: March 27, 2026*

**Status:** Working canon  
**Layer:** Ground 4 / Grammar-Architecture  
**Authority:** Development control  
**Scope:** Defines the purpose and structure of the Grammar-Architecture folder within Varoli Ground 4.

---

## 1. Purpose

This folder contains the **higher-order grammar architecture** of Varoli.

It is the part of Ground 4 that formalizes the structural categories, relations, and large-scale compositional behaviors that arise from the primitive foundation established in Ground 3.

This folder does not define the primitive inventory itself.
It depends on that inventory and explains what becomes possible when primitive structure scales into larger compositional units.

---

## 2. Role of this folder

The Grammar-Architecture folder exists to hold developments such as:

- containment
- qualification
- framing scope
- perspective markers
- stable-form expansion
- resolution-unit chaining
- verb layering
- negation-adjacent architecture
- other higher-order structural categories derived from the primitive base

These are not raw primitives.
They are structural developments that describe how primitive forms behave in larger systems.

---

## 3. What belongs here

A document belongs in this folder if it is:

- higher-order than primitive generation
- dependent on Ground 3 inventories and interaction rules
- part of the language’s grammar architecture rather than its raw primitive map
- important enough to guide downstream work
- not specifically a Yemu proof-object or simultaneous-expression file

This folder is for:
- structural relations
- compositional laws
- large-scale grammar behaviors
- derivable higher-order categories

It is not the place for:
- primitive reference files
- raw directional-form inventories
- primitive stable-world inventories
- loose exploratory fragments without structural grounding

---

## 4. Relation to Ground 3

This folder depends directly on Ground 3.

Ground 3 provides:
- primitive building blocks
- primitive directional forms
- primitive stable worlds
- primitive generation logic

The documents in this folder should therefore be read as **derived from Ground 3**, not as independent law.

If a Grammar-Architecture claim cannot be reconciled with:
- primitive class constraints
- primitive interaction rules
- directional-form logic
- primitive stable-world logic

then it should be revised, downgraded, or rejected.

---

## 5. Relation to Yemu

Ground 4 also contains Yemu-specific work, but Yemu should not be collapsed into the Grammar-Architecture folder.

This folder is for the higher-order grammar architecture of Varoli broadly.

Yemu, self-crucible, simultaneous expression mode, and related proof-object work should live in their own Ground 4 area because they are not simply grammar extensions. They are a distinct higher-order expression and compression domain.

---

## 6. Current status rule

Files in this folder should be treated as serious structural development.

Some may function as working canon.
Others may remain provisional structural notes.
Their exact status should be read from each file’s own header.

This folder is allowed to develop actively, but all development here remains accountable to:
- Ground 0
- Ground 1
- Ground 2
- Ground 3

---

## 7. How this folder should be used

This folder should be used to:

- formalize higher-order structural discoveries
- define major grammar categories in native terms
- explain how primitive forms scale into larger structure
- support later application, conversation, and narrative development
- provide a stable architecture layer above the primitive foundation

This folder should not become:
- a duplicate of Ground 3
- a general notes dump
- a replacement for Yemu-specific documentation
- an imported conventional grammar framework

---

## 8. Bottom line

Ground 3 provides the generative foundation of Varoli.

This folder develops the higher-order grammar architecture that emerges from that foundation.
