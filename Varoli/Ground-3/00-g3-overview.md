# 00 — Varoli Ground 3 Overview

*Last Updated: March 28, 2026*

**Status:** Working reference  
**Layer:** Ground 3  
**Authority:** Development control  
**Scope:** Defines the purpose and structure of the Ground 3 generative foundation layer.

---

## 1. Purpose

Ground 3 contains the **generative foundation** of Varoli.

This is the layer where the language is made buildable, searchable, learnable, and usable from its immediate structural materials.

Ground 3 does not primarily formalize higher-order grammar architecture.
Instead, it establishes the reference layers from which higher-order structure can later be derived, tested, and stabilized.

---

## 2. Role of Ground 3

Ground 3 exists to hold the language’s immediate generative base:

- primitive building blocks
- primitive directional forms
- primitive stable worlds
- closely related exploratory work tied directly to generation, learnability, and readable use

This is the first layer where the language becomes operational rather than only conceptual.

---

## 3. What belongs here

A document belongs in Ground 3 if it is:

- part of the primitive generative foundation
- directly usable for building or reading forms
- necessary for learnability and reference
- still below the level of higher-order grammar architecture

Ground 3 is for:
- inventories
- schemas
- interaction rules
- primitive maps
- stable-world maps
- tightly related exploratory notes

Ground 3 is not the main home of advanced grammar interpretation.

---

## 4. Current folder structure

### `Primitive-Building-Blocks/`
The primitive inventory and interaction foundation.

This contains:
- primitive overview
- vowels
- operator
- consonants
- interaction rules
- primitive schemas

### `Primitive-Directional-Forms/`
The primitive directional inventory.

This contains the two-character directional forms generated from:
- CV
- VC

organized for learnability and generation.

### `Primitive-Stable-Worlds/`
The primitive stable-world inventory.

This contains:
- stable-world overview
- same-envelope primitive stable worlds
- cross-envelope placeholder work
- supporting notes

### `Exploring/`
The immediate frontier tied directly to Ground 3 growth.

This contains:
- next-step exploratory notes
- unresolved primitive-generation pressures
- ideas not yet ready for formal higher-order treatment

---

## 5. Relation to Ground 4

Ground 3 is the foundation that Ground 4 depends on.

Ground 4 contains:
- higher-order structural development
- grammar architecture
- Yemu
- self-crucible
- advanced stable-form work
- chaining, qualification, containment, negation, and related developments

Ground 3 should therefore be read before Ground 4 whenever a higher-order development depends on primitive structure.

---

## 6. Current status rule

Ground 3 files should be treated as serious working reference.

They are meant to be:
- compact
- stable enough to build from
- revisable if higher-layer conflict or later derivation requires correction

This layer favors:
- scanability
- consistency
- generation support
- low-drift reference structure

---

## 7. How Ground 3 should be used

Ground 3 should be used to:

- learn the primitive system
- generate primitive forms
- compare substrate and center behavior
- identify promising recurring worlds
- support conversation and storytelling experiments
- provide clean reference support for Ground 4 work

Ground 3 should not become:
- a second conceptual overview
- a prose-heavy theory layer
- a duplicate of Ground 4 grammar documents

---

## 8. Bottom line

Ground 2 explains what Varoli is.

Ground 3 provides the primitive generative foundation of the language.

Ground 4 develops the higher-order structures that depend on that foundation.
