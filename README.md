# ICA / Varoli Repository

This repository is the working home of the **Invariant Compositional Architecture (ICA)** and of **Varoli**, its human-language proof of implementation.

It exists to support two closely related aims:

1. to document the architecture clearly enough that it can be studied without drift  
2. to develop Varoli as a living language without losing the precision that makes its growth coherent

Varoli did not begin as a system diagram. It began as a language encounter, and over time that encounter revealed an architecture beneath it. This repository reflects that discovery. It is structured so the architecture can be studied in its own right while the language remains human, expressive, and alive at its center.

The project is still under active development. That means this repository is not only a record of settled structure. It is also a workspace for ongoing derivation, clarification, and testing. Some layers are stable. Others remain open where the language is still teaching us what it is.

## How to read this repository

The repository is organized into **Grounds** so that different kinds of knowledge remain distinct.

Each Ground has a different role. Together, they show how the project moves:
- from abstract architecture
- into adaptation
- into Varoli's own self-understanding
- into the immediate building materials of the language
- and then into higher-order structural development

This structure helps preserve both exactness and discovery. It allows the language to grow without collapsing its own foundations.

## Repository structure

### `Ground-0/`
The abstract ICA layer.

This contains the domain-agnostic architectural foundation:
- terminology
- blueprint
- scaling logic

Ground 0 defines the architecture itself. It does not define Varoli directly. It is the most abstract layer in the repository, and it should be read as architecture, not as language.

### `Varoli/`
The Varoli implementation layer.

This is where ICA becomes a specific language system. Varoli is organized in layered form so the relationship between architecture, adaptation, orientation, generation, and later structural development remains visible.

#### `Ground-1/`
The adaptation layer.

This explains how ICA becomes Varoli without reducing Varoli to ICA vocabulary. It is the bridge between the abstract architecture and the language as a language.

Ground 1 matters because Varoli is not a decorative example of ICA. It is the originating implementation through which the architecture became visible.

#### `Ground-2/`
The Varoli-native orientation layer.

This establishes Varoli in its own terms:
- terminology
- conceptual overview

Ground 2 is where the language begins to speak from itself. It gives the reader the conceptual orientation needed to understand what Varoli is trying to do and how it should be approached.

If Ground 1 explains the transition into Varoli, Ground 2 helps the reader meet Varoli on its own ground.

#### `Ground-3/`
The generative foundation layer.

This is where the language is built from its immediate structural materials:
- primitive building blocks
- primitive directional forms
- primitive stable worlds
- closely related exploratory work tied directly to generation and learnability

Ground 3 is the first place where Varoli becomes truly usable as a language foundation. It contains the inventory, the first valid constructions, and the nearest structural materials from which larger formations can emerge.

For a reader who wants to learn the language as it develops, this is often the most important Ground. It is where structure begins to feel alive.

#### `Ground-4/`
The higher-order structural development layer.

This is where larger structural discoveries and grammar architecture are formalized after the generative foundation is in place. Ground 4 includes:
- named structural relations
- chaining architecture
- qualification and containment
- stable-form expansion
- negation-adjacent architecture
- other higher-order developments derived from and dependent on Ground 3

Ground 4 is not loose speculation, but it is also not treated as untouchable. It is a developing layer where later discoveries may still sharpen, revise, or reorganize what has been formalized.

This is the layer where the language begins to show the consequences of its deeper architecture.

## Development principle

This repository is organized to preserve **layer integrity**:

- **Ground 0** stays abstract
- **Ground 1** bridges ICA into Varoli
- **Ground 2** speaks in Varoli's own terms
- **Ground 3** provides the generative foundation of the language
- **Ground 4** develops higher-order structure without rewriting the layers above it

The purpose of this separation is not bureaucracy. It is protection.

Without clear layers, a project like this can drift in several directions at once:
- architecture can be mistaken for language
- descriptive language can be mistaken for structure
- exploratory ideas can be mistaken for canon
- later discoveries can overwrite first principles instead of deriving from them

The Ground structure exists to prevent that drift while still allowing discovery to continue.

## What kind of project this is

This repository is not only a grammar reference, and it is not only a research archive.

It is a developmental workspace for:
- a language
- the architecture discovered beneath that language
- and the ongoing effort to preserve both precision and aliveness as the system grows

Some documents are strictly structural. Some are orienting. Some remain exploratory. Together they show the project in motion.

If you are here to study the architecture, the Grounds will help you track the logic cleanly.

If you are here to learn Varoli, the Grounds will help you see how the language is being formed from first principles rather than assembled from inherited categories.

If you are here because language, structure, and human meaning should not have to separate from one another, that is part of what this repository is trying to understand.
