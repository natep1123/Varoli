### Different tested domains all fail to bridge for related but distinct reasons

**Not single missing element but multiple architectural gaps:**

**Gap A (Epistemic/Ontological):**

Derivability and underivability are epistemic (about what can be derived).

Status categories are ontological (about what forms are).

Tested priorities attempting epistemic → ontological bridge: 1 (rule-language), 4 (compositional capacity).

Both failed: Epistemic properties don't mechanically determine ontological status.

**Gap B (Necessary/Sufficient):**

Necessary conditions identified (primitive preservation, conditional rules).

Sufficient conditions not derivable.

Tested priorities attempting sufficiency: 2 (irreducible core), 3 (primitive functions).

Both failed: Necessary ≠ sufficient without proof.

**Gap C (Definitional/Compositional):**

Primitives define what elements are (vowels = substrate, consonants = kinetic).

Rules describe how elements relate compositionally.

Tested priorities examining both levels: 3 (primitives), 1-2-4 (rules/patterns/composition).

Primitives provide only definitional constraint (non-contradiction).

Compositional rules provide only derivability constraints or remain ambiguous.

**Implication:** Multiple architectural elements would be needed to bridge fully, not just one missing piece.

## 4. Supported Provisional Findings

### Finding 1: Primitive Function Violation Provides Mechanically Clearer Invalidity Bridge via Contradiction and Non-Generation

**Primitive function violation defined:**

Using primitive of class X with function F_y where F_x ≠ F_y

**Example:** Vowel (substrate-defining function) used as kinetic operator

**Bridge mechanism:**

Primitive contradiction → Non-generatable from primitives → Invalid

**Why mechanically clearer:**

1. **Creates contradiction:** F_x AND NOT F_x (logical impossibility)
2. **Prevents generation:** Cannot generate from primitives while preserving invariant functions
3. **Definitional invalidity:** Non-generatable = not a Varoli form

**Mechanical derivation:**
- Premise: Varoli forms are generated from primitives with invariant functions (foundational)
- Consequence: Forms contradicting primitive functions cannot be generated
- Status: Invalid (definitionally, not a Varoli form)

**Scope:** Primitive-level contradictions only

**Status:** Strongly supported by mechanical analysis

---

### Finding 2: Primitive Contradiction vs. Rule Unsatisfiability Is Key Distinction

**Primitive function violation:**
- Creates **contradiction** (definitional impossibility)
- Mechanism: F_x AND NOT F_x at primitive level
- Consequence: Non-generatable from primitives
- Bridges to: Invalid status (mechanically clear)

**Rule-level violation:**
- Creates **unsatisfiability** or **unmet requirement** (operational ambiguity)
- Mechanism: Operation cannot be satisfied OR requirement not met
- Consequence: Primitives preserved, form generatable
- Bridges to: No mechanically derived status (gap persists)

**Fundamental difference:**

Primitives are definitional (what elements ARE)
- Contradiction at this level = impossibility
- Non-generation is mechanical consequence

Rules are operational (how elements RELATE)
- Violation at this level = ambiguity/unsatisfiability
- Generation still possible (primitives preserved)

**Status:** Central distinction of Stage 11

---

### Finding 3: Rule 3 Failure Creates Unsatisfiability, Not Contradiction

**Test case:** `o + VCV + CV` (CV cannot bind forward, no following VCV)

**Analysis:**

**Rule 3 requires:** CV binds forward to next VCV

**In test form:**
- CV's primitive function preserved (kinetic → substrate directionality)
- Forward binding operation unsatisfiable (no target available)
- Form generatable from primitives

**Mechanism:**

Unsatisfiable operation (like function with no valid input)

NOT contradiction (like function required to be both X and NOT X)

**Consequence:**

Unsatisfiability → Underivability (compositional relation unclear)

NOT: Unsatisfiability → Invalidity (no contradiction, no bridge)

**Status:** Rule 3 failure does not bridge to invalidity mechanically

### Finding 4: Rule 5 Failure Creates Unmet Requirement, Not Demonstrable Contradiction

**Possible interpretations:**

**Prohibition reading:** Centers cannot exist without mediation → Contradiction → Invalid

**Derivability reading:** Centers need mediation for derivable relation → Unmet derivability condition → Underivable

**Mechanical support:**

Rule 5 states "require" but not consequence.

Rule 4 context ("not derivable") suggests derivability interpretation.

Prohibition interpretation possible but not mechanically forced.

**Consequence:**

Unmet requirement → Mechanically undeterminable status (ambiguity persists)

**Status:** Rule 5 failure does not demonstrably bridge to invalidity mechanically
